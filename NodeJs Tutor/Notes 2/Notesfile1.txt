Synchronous and Asynchronous Tasks
    Java script is a Synchronous Programming language.
    Lekin jab baat Node Js k sath ho to JavaScript Asynchronous or Non-Blockig Programming language ban jati he.

    Synchronous ==  When inital or any one task not complete during this period we will not move on next step that is
    called Synchronous Programming.

    Asynchronous == When inital or any one task not complete during this period we will move on next step that is
    called Synchronous Programming. All task excute togather

Synchronous Proccessing
JavaScript file ka execution top to bottom hota he. Jaise hi ek file ka execution start hota he memory me ek entity ban
k aati he jise call stack kehte he. call stack ek storage area hota he jo data ko stack ki form me store karta he. stack
follow karta he LIFO ya FILO concept ko.
Execution start hote hi sabse pehle Global execution context call stack me push ho jata he.


Asynchronous Proccessing
JavaScript file ka execution top to bottom hota he. Jaise hi ek file ka execution start hota he memory me ek entity ban
k aati he jise call stack kehte he. Execution start hote hi sabse pehle Global execution context call stack me push ho
jata he. Sath me 2 entity ban k aati he jisme event loop or callback queue kehte he. queue follow karta he FIFO concept.

Event loop ka kaam hota he timer pr nazar rakhna or callstack k empty hone ka wait karna. agar call stack epmty hota he
to callback queue se function ko nikal kar call stack me dalna.



Why do we need Asynchronous Execution in JS?
JavaScript internally single threaded (single worker) programming language he.
